Data Types and Variables in C

C requires explicit type declarations for variables. This allows the compiler to allocate appropriate memory and perform type checking, leading to efficient code.

Basic Data Types

Choose types based on the range and precision needed. Using the wrong type can cause overflow or precision loss.

TypeSize (bytes)RangeUse Case
char1-128 to 127Single characters, small integers
short2-32,768 to 32,767Small integers
int4-2^31 to 2^31-1General integers
long8-2^63 to 2^63-1Large integers
float4~6 decimal digitsFloating point, less precision
double8~15 decimal digitsFloating point, more precision

Advanced Note: Sizes can vary by platform. Use sizeof() to check. For portability, prefer int for integers unless size matters.

Variables

Variables must be declared before use. Declaration tells the compiler the type and allocates memory.

int age = 25;  // Integer variable
float pi = 3.14;  // Floating point
char letter = 'A';  // Character

Uninitialized variables contain garbage values. Always initialize for predictable behavior.

Constants

Constants cannot be changed after initialization. Use for fixed values.

const int MAX_SIZE = 100;  // Runtime constant
#define PI 3.14159  // Compile-time constant

Key Difference: const is checked at compile time, #define is a preprocessor replacement.

Type Casting

Convert between types explicitly. Implicit casting can lose data.

int x = 5;
double y = (double)x;  // Explicit cast

Advanced Note: Casting pointers can be dangerous. Use with care to avoid undefined behavior.
