Preprocessor Directives in C

The preprocessor runs before compilation. It handles includes, defines, and conditional compilation.

#include

Inserts contents of header files. Use <> for system headers, "" for local.

#include <stdio.h>    // Standard library
#include "myheader.h"  // Custom header

Headers declare functions/variables. Source files define them.

#define

Text substitution. No type checking.

#define PI 3.14159
#define MAX(a,b) ((a) > (b) ? (a) : (b))

Use parentheses in macros to avoid precedence issues.

Conditional Compilation

#ifdef DEBUG
    printf("Debug mode\n");
#endif

#ifndef HEADER_H
#define HEADER_H
// Header guards prevent multiple inclusion
#endif

Useful for platform-specific code or debug builds.

#if, #else, #elif

#if DEBUG == 1
    printf("Debug mode\n");
#endif

Allows complex conditions with defined() checks.

Other Directives

  • #pragma - Compiler-specific instructions
  • #error - Stop compilation with message
  • #line - Change reported line number
  • #undef - Remove macro definition

Predefined Macros

  • __FILE__ - Current file name
  • __LINE__ - Current line number
  • __DATE__ - Compilation date

Advanced Note: Preprocessor is powerful but can make code hard to debug. Use const for constants when possible.
