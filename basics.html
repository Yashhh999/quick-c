Basic Syntax and Concepts in C

C is a compiled language where you write source code in .c files, compile it to machine code, and run the executable. Unlike interpreted languages, this gives high performance but requires compilation.

Program Structure

Every C program starts with the main function. This is the entry point where execution begins. The return 0; indicates successful completion.

#include <stdio.h>  // Includes standard input/output functions

int main() {  // Function definition: return type, name, parameters
    // Statements go here
    return 0;  // Return value to the operating system
}

Key Points: Functions must declare return types. main typically returns int. The #include directive brings in external code.

Comments

Comments are ignored by the compiler and used for documentation.

// Single line comment - for brief notes
/* Multi-line comment
   for longer explanations */

Use comments to explain complex logic or assumptions.

Semicolons and Statements

Each statement ends with a semicolon. This tells the compiler where one instruction ends and another begins.

printf("Hello");  // Statement 1
printf("World");  // Statement 2

Forgetting semicolons is a common compilation error.

Case Sensitivity

C is case-sensitive: main and Main are different identifiers.

Whitespace

Whitespace (spaces, tabs, newlines) is ignored except in strings. Use it for readability.

int x=5;  // Valid but hard to read
int y = 5;  // Better

Basic Input/Output

printf outputs formatted text to the console. scanf reads input.

#include <stdio.h>

int main() {
    printf("Hello, World!\n");  // \n is newline
    return 0;
}

Advanced Note: printf uses format specifiers like %d for integers, %f for floats. Always check return values for errors in production code.
