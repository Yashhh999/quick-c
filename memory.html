Dynamic Memory Management in C

C requires manual memory management. Allocate when needed, free when done. Mistakes cause leaks or crashes.

malloc

Allocates uninitialized memory. Returns void* (cast to type).

int *arr = (int*)malloc(5 * sizeof(int));

Check for NULL return. Memory may contain garbage.

calloc

Allocates and initializes to zero. Slower but safer.

int *arr = (int*)calloc(5, sizeof(int));

realloc

Resize existing allocation. May move data.

arr = (int*)realloc(arr, 10 * sizeof(int));

If fails, original pointer unchanged. Copy data if needed.

free

free(arr);

Deallocates memory. Accessing freed memory is undefined. Double free crashes.

Memory Leaks

Forgetting to free causes leaks. Programs consume more memory over time.

// Leak
int *p = malloc(sizeof(int));
// Use p
// Forgot free(p);

Common Mistakes

  • Dangling pointers: Using freed memory
  • Double free: Freeing already freed pointer
  • Buffer overflow: Accessing beyond allocated memory
  • Use after free: Accessing freed memory

Tools

Use Valgrind to detect memory issues. Compile with -fsanitize=address for runtime checks.

Advanced Note: For complex programs, use smart pointers or garbage collectors, but C doesn't have them built-in.
